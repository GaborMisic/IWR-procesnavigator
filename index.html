01

Wat doet deze tool

A

Afhankelijkheden bewaken

Per project één rij per onderzoek, vergunning, boring, proefgat, verkeersplan, schouw of gate. Status, eigenaar, normdoorlooptijd en buffer — alles zichtbaar.

B

GSU-voorspelling

Per taak een geïmpliceerde GSU: de vroegst haalbare startdatum die dit item toelaat. Het projectoverzicht toont de MAX daarvan = de verwachte GSU.

C

Kritiek pad aanwijzen

Welk onderzoek, welke vergunning, welke gate veroorzaakt de slip? Eén kolom, één naam. Dat is het interventiepunt voor de engineer.

02

Hoe de voorspelling werkt

Vereiste gereed-datum per item WORKDAY(GSU; –Buffer_Wd)
Uiterlijke uitzetdatum WORKDAY(Vereist_Gereed; –Norm_Doorlooptijd_Wd)
Verwacht gereed WORKDAY(Datum_Uitgezet; +Norm_Doorlooptijd_Wd)
Geïmpliceerde GSU per item WORKDAY(Verwacht_Gereed; +Buffer_Wd)
Verwachte GSU per project MAX(Geimpliceerde_GSU) over items van dit project

Eenheid overal in werkdagen. Weekenden worden automatisch overgeslagen door WORKDAY en NETWORKDAYS. De tool onderscheidt route × variant (KG / NG / RC × NOidK / OidK) en gebruikt per combinatie de juiste buffer uit het IWR-contract.

03

Werkwijze in 4 stappen

  1. 1

    Project aanmaken

    Vul in blad Projecten: ProjectID, Projectnaam, GSU-datum, Route, Variant, Engineer.

  2. 2

    Afhankelijkheden toevoegen

    In Dashboard_Engineer: één rij per onderzoek/vergunning/gate. Dropdowns cascaderen: Hoofdcategorie → Subcategorie → Type.

  3. 3

    Status bijhouden

    Zet status op Ingediend → datum wordt gestempeld (met VBA). Bij Afgerond → ontvangen-datum. Stoplicht wordt vanzelf groen/oranje/rood.

  4. 4

    Projectoverzicht lezen

    Hier staat de verwachte GSU per project, slip in werkdagen/weken, en welk item het kritieke pad bepaalt.
